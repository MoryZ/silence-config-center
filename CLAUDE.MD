# CLAUDE.md

## 项目开发规范（silence-config-center）

本文档基于当前仓库代码结构与实现方式提炼，作为后续开发与代码评审基线。

## 1. 架构模式

### 1.1 分层结构
- `api` 层（Controller/Resource）：负责 HTTP 入参接收、参数转查询条件、DTO 到领域对象转换，不直接写 SQL。
  - 参考：`src/main/java/com/old/silence/config/center/api/ConfigItemResource.java`
  - 参考：`src/main/java/com/old/silence/config/center/api/ConfigAccessKeysResource.java`
- `domain/service` 层：承载业务编排与领域规则（克隆、同步、签名校验、长轮询通知等）。
  - 参考：`src/main/java/com/old/silence/config/center/domain/service/ConfigNamespaceService.java`
  - 参考：`src/main/java/com/old/silence/config/center/domain/service/LongPollingService.java`
- `domain/repository` 层：定义仓储接口，隔离业务与具体 ORM/SQL 实现。
  - 参考：`src/main/java/com/old/silence/config/center/domain/repository/ConfigItemRepository.java`
- `infrastructure/persistence` 层：仓储接口实现，组合 MyBatis-Plus `BaseMapper` 与必要自定义 SQL。
  - 参考：`src/main/java/com/old/silence/config/center/infrastructure/persistence/ConfigItemMyBatisRepository.java`
- `infrastructure/persistence/dao` 层：MyBatis Mapper（DAO），承载 SQL 细节。
  - 参考：`src/main/java/com/old/silence/config/center/infrastructure/persistence/dao/ConfigItemDao.java`

### 1.2 职责边界约束
- Controller 仅做协议层职责：
  - 查询条件构建使用 `QueryWrapperConverter`。
  - DTO 与领域模型转换使用 MapStruct（`api/assembler`）。
- 业务逻辑进入 Service 或 Repository：
  - 例如发布后通知逻辑在仓储实现中集中处理，而非 Controller 内拼接。
  - 参考：`src/main/java/com/old/silence/config/center/infrastructure/persistence/ConfigItemReleaseHistoryMyBatisRepository.java`
- Repository 接口面向领域语义，DAO 面向 SQL 语义。

## 2. 数据库规范

### 2.1 表命名规则
- 当前项目表名统一为 `snake_case`，并使用业务域前缀 `config_`。
  - 示例：`config_item`、`config_environment`、`config_component`、`config_access_keys`
  - 参考：`src/main/java/com/old/silence/config/center/infrastructure/persistence/dao/ConfigItemDao.java`
  - 参考：`src/main/java/com/old/silence/config/center/infrastructure/persistence/dao/ConfigAccessKeysDao.java`
- 本仓库未采用 `t_` 前缀；新增表应延续 `config_` 域前缀与语义化命名。

### 2.2 主键与审计字段
- 实体统一继承 `AbstractAutoGeneratedIdAuditable<BigInteger>`，主键类型为 `BigInteger`。
  - 参考：`src/main/java/com/old/silence/config/center/domain/model/ConfigItem.java`
  - 参考：`src/main/java/com/old/silence/config/center/domain/model/ConfigEnvironment.java`
- 审计字段由基类统一托管（常见为创建/修改时间、创建/修改人）。
  - 虽未在本仓库实体中逐一显式声明，但为统一审计基线，建表时必须保证审计字段可自动填充（通常对应 `gmt_create`、`gmt_modified` 一类字段）。

### 2.3 字段命名与映射
- Java 字段使用 `camelCase`，数据库列使用 `snake_case`。
- 布尔开关列采用 `is_xxx`，并通过 `@TableField` 显式映射。
  - 示例：`enabled -> is_enabled`，`display -> is_display`
  - 参考：`src/main/java/com/old/silence/config/center/domain/model/ConfigAccessKeys.java`
  - 参考：`src/main/java/com/old/silence/config/center/domain/model/ConfigEnvironment.java`

### 2.4 JSON 字段处理方式
- 当前未发现数据库 JSON 列及 `typeHandler`（如 `JacksonTypeHandler`）使用。
- 当前 JSON 处理主要在接口响应层：
  - `ObjectMapper` 输出 JSON。
  - 对 `BigInteger` 等字段使用 `@JsonFormat(shape = JsonFormat.Shape.STRING)` 防止前端精度丢失。
  - 参考：`src/main/java/com/old/silence/config/center/vo/ConfigItemVo.java`
- 若后续引入 JSON 列：必须显式指定 `typeHandler`，禁止手工字符串拼接 JSON 入库。

## 3. 常用工具栈

### 3.1 持久化与查询
- MyBatis-Plus（`platform-data-mybatis-plus`）
  - 场景：`BaseMapper` 基础 CRUD、`QueryWrapper/LambdaQueryWrapper/UpdateWrapper` 条件构建、`Page` 分页。
  - 参考：`src/main/java/com/old/silence/config/center/infrastructure/persistence/ConfigItemMyBatisRepository.java`

### 3.2 对象转换
- MapStruct + mapstruct-spring
  - 场景：`Command/Query` 与领域对象转换，减少手写转换样板代码。
  - 参考：`src/main/java/com/old/silence/config/center/api/assembler/ConfigItemMapper.java`

### 3.3 Web 与安全
- Spring Boot Web + Security
  - 场景：REST API、拦截器签名鉴权（`SignatureAuthInterceptor`）。
  - 参考：`src/main/java/com/old/silence/config/center/api/config/SignatureAuthInterceptor.java`

### 3.4 配置与注册
- Nacos（discovery + client）
  - 场景：服务发现与配置中心生态集成。
  - 依赖定义：`pom.xml`

### 3.5 缓存
- Caffeine
  - 场景：本地高性能缓存（依赖已引入，优先用于本地短时缓存与热点数据）。
  - 依赖定义：`pom.xml`

### 3.6 其他
- MySQL Connector/J：数据库驱动。
- Apache Commons Lang3：字符串与通用工具。
- 当前仓库未发现 Hutool/Redisson 依赖与使用代码。

## 4. 开发红线

以下红线由现有异常处理与日志风格抽取，作为强约束执行：

1. 禁止吞异常。
- 禁止空 `catch` 或仅注释“忽略异常”。
- 必须至少记录 `error` 日志并带异常栈，或转换为统一业务异常抛出。
- 反例位置：`src/main/java/com/old/silence/config/center/domain/service/LongPollingService.java`

2. 禁止 `e.printStackTrace()`。
- 一律使用统一日志框架（SLF4J）输出。
- 反例位置：`src/main/java/com/old/silence/config/center/domain/service/LongPollingService.java`

3. 禁止在循环中逐条查库（N+1）。
- 在批量场景中应优先批量查询、批量写入。
- 风险位置：`sync/clone` 流程中按目标项逐次调用仓储。
- 参考：`src/main/java/com/old/silence/config/center/domain/service/ConfigNamespaceService.java`

4. 禁止记录敏感信息明文。
- 认证/签名相关日志不可输出完整 `signature/secretKey/nonce`。
- 需脱敏或仅记录必要片段。
- 风险位置：`src/main/java/com/old/silence/config/center/api/config/SignatureAuthInterceptor.java`

5. 禁止跳过统一错误语义。
- 参数错误、权限错误、数据不存在等应使用统一错误工厂（`CommonErrors`）返回可观测、可治理的错误。
- 正例：`CommonErrors.INVALID_PARAMETER`、`CommonErrors.DATA_NOT_EXIST` 使用方式。

6. 禁止在 Controller 写业务编排与 SQL。
- Controller 仅处理协议与转换，不拼 SQL、不做跨聚合业务事务。
- 仓储层负责数据访问细节，Service 负责业务编排。

## 5. 建议的评审清单（PR Checklist）

- 是否遵守 `api -> domain -> infrastructure` 单向依赖。
- 是否存在循环查库、循环更新未批处理问题。
- 是否存在吞异常、`printStackTrace`、日志无上下文的问题。
- 是否出现敏感信息日志泄露。
- 新增实体是否沿用 `BigInteger` 主键与审计字段约束。
- 新增数据库字段是否符合 `snake_case` 命名及显式映射规则。
- 新增 DTO/Command 是否通过 MapStruct 转换，避免手写重复转换代码。
